[WORKFLOW]
FlowZint workflow automation allows businesses to build, schedule, and monitor automated processes without manual intervention.
Workflows are built using a visual drag-and-drop builder with trigger nodes, condition nodes, action nodes, and output nodes.
Supported trigger types: Webhook (HTTPS only), Scheduled (cron-based UTC), API Event, Form Submission, and Manual Run.
If a workflow fails, navigate to Workflows > Execution History to view the failed step, error message, and timestamp.
Workflow execution timeout limits: Starter plan 5 minutes, Growth plan 15 minutes, Enterprise plan unlimited.
To debug a stuck workflow: disable the workflow, inspect the last execution log, fix the failing node, re-enable and test.
Webhook triggers require HTTPS endpoints. HTTP endpoints are rejected for security compliance.
Failed workflows do not auto-retry by default. Enable retry logic under Workflow Settings > Error Handling > Retry Policy.
Scheduled workflows run in UTC timezone. Adjust for IST by adding +5:30 offset in the schedule configuration.
For workflow execution delays exceeding 10 minutes, contact FlowZint support at contact@flowzint.in with your workflow ID.
Maximum concurrent workflows: Starter 5, Growth 25, Enterprise unlimited.

[API]
FlowZint API base URL: https://api.flowzint.in/v1/
Generate API keys at Settings > Developer > API Keys. Each key has configurable permissions, IP restrictions, and expiry dates.
API authentication uses Bearer token format. Include in request header: Authorization: Bearer YOUR_API_KEY
Common HTTP error codes: 401 (invalid or expired API key), 403 (insufficient permissions), 429 (rate limit exceeded), 500 (platform error).
Rate limits by plan: Starter 100 requests/minute, Growth 1000 requests/minute, Enterprise custom negotiated limits.
Webhook payloads are sent as JSON POST requests. Your endpoint must return HTTP 200 within 5 seconds or delivery is retried.
Webhook retry policy: 3 attempts at 5-minute intervals, then marked as failed.
FlowZint SDK available for Python, Node.js, and Go.
Python install: pip install flowzint-sdk
Node.js install: npm install @flowzint/sdk
OAuth 2.0 supported for third-party application integrations. Initiate flow at /auth/oauth/authorize endpoint.
API documentation and interactive playground available at https://flowzint.in — contact support for API docs access.
For API outages or platform-side errors, contact contact@flowzint.in with request ID and HTTP response body.
IP whitelisting for API keys available on Growth and Enterprise plans under Settings > Developer > API Security.
